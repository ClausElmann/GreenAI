# UI Auto-Fix Protocol

**Status:** ACTIVE  
**Last Updated:** 2026-04-05  
**Trigger words:** `fix ui`, `run ui autofix`, `repair ui`, `ui broken`, `visual issues`, `accessibility issues`  
**Script:** `scripts/run-ui-autofix.ps1`

---

## PURPOSE

Automatically detect and fix UI quality violations using the E2E quality gate system.  
The loop runs until all gates pass or the iteration limit is reached.

Quality gates (run per device on every visual test):
- **Axe-core WCAG 2.0 A+AA** — contrast, keyboard nav, document structure
- **Design tokens** — all `--ga-*` CSS custom properties defined and correct
- **Focus ring** — outline-width ≥ 2px after Tab keypress
- **Touch targets** — ≥ 44×44px on mobile (WCAG 2.5.5)
- **Content padding** — safe horizontal padding on main content containers

---

## INPUT

Failures are sourced from E2E test output. The file `TestResults/Visual/ui-failures.json`
is generated by the loop — it does **not** need to exist before running.

---

## EXECUTION FLOW

```
1. Set GREENAI_ACCESSIBILITY_GATES=true
2. Run E2E visual tests
3. Parse failures from test output
4. Write TestResults/Visual/ui-failures.json
5. Apply deterministic fix strategies (see table below)
6. Rebuild
7. Re-run tests
8. Repeat (max 3 iterations)
9. Report results
```

---

## FIX STRATEGIES

| Failure type     | Fix strategy                                             |
|------------------|----------------------------------------------------------|
| `contrast`       | Override with `--ga-*` colour token via `!important`; if token doesn't reach the element, add inline `<style>` override after `<MudThemeProvider />` in layout files |
| `spacing`        | Apply `--ga-space-*` tokens (pad/margin)                 |
| `touch-target`   | `min-height: 44px` on `.mud-button-root` / `.mud-icon-button.mud-button-root` inside `@media (max-width: 768px)` in `greenai-skin.css` |
| `focus`          | Add `:focus-visible` outline rule to component CSS or skin |
| `overflow`       | Add `padding` or `overflow-x: hidden` to layout container |
| `raw-label`      | Replace hardcoded string with `@Loc.Get("...")` call     |
| `overlap`        | Adjust `z-index` or `position` in component CSS          |
| `document-title` | Add `<title>` fallback in `App.razor` or `<PageTitle>` in layout |

---

## RULES

- No redesign allowed — structural changes require user approval
- No layout restructuring — only targeted property fixes
- Only deterministic fixes — each failure type maps to exactly one fix strategy
- Max 3 iterations before stopping and reporting remaining failures
- Build must remain clean: `0 errors, 0 warnings` after each fix
- Never add inline `style=""` attributes on Razor elements
- CSS fixes go in `greenai-skin.css` (global) or component `.razor.css` (scoped)

---

## KNOWN GOTCHAS

### MudThemeProvider overrides palette variables
`<MudThemeProvider />` injects default pink palette (`--mud-palette-secondary: rgba(255,64,129,1)`)
into the DOM **at runtime** — after `greenai-skin.css` loads. This overrides our `:root` skin tokens.

**Fix:** Add `<style id="greenai-palette-override">` block immediately after `<MudThemeProvider />`
in every layout file (`MainLayout.razor`, `WizardLayout.razor`).  
See [MainLayout.razor](../../src/GreenAi.Api/Components/Layout/MainLayout.razor).

### CSS class specificity for icon buttons
`.mud-icon-button` has a global `min-height: unset !important` rule. The touch-target media-query
override must use `.mud-icon-button.mud-button-root` (two classes) for equal specificity + later cascade win.

### Scoped Blazor CSS requires rebuild
Changes to `.razor.css` files only take effect after `dotnet build`. Changes to files under
`wwwroot/` are served immediately from source (no rebuild needed).

## OUTPUT

```
1. Iterations performed        (max 3)
2. Files modified              (list)
3. Fixes applied               (type + selector + file)
4. Remaining failures          (if any after 3 iterations)
5. Final test count            (Passed / Failed)
```
